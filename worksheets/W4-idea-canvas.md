## 🎯 1. Problem & User

### a) เราจะแก้ปัญหาอะไร?
```
เกษตรกรใช้สายตาเเละความรู้สึกในการประเมินหน้างานทำให้เกิดความคลาดเคลื่อนสูง
เเนวทางการดูเเลยังไม่มีประสิทธิภาพเนื่องจากรูปเเบบการวิเคราะห์อาศัยการประเมินเชิงคาดการณ์เป้นหลัก
การหลงเชื่อข้อมูลที่ผิดพลาดหรือคำโฆษณาเกินจริงของผลิตภัณฆ์
```

### b) ใครคือ user หลัก?
```
-ชาวสวนเเละเกษตรกรทุเรียนที่เข้าไม่ถึงคำเเนะนำที่ถูกต้อง
-ชาวสวนทุเรียนมือใหม่ที่ขาดประสบการณ์
-กลุ่มชาวสวนทุเรียนที่เผชิญปัญหา ทุเรียนไม่ยอมออกดอก ออกดอกน้อย ผลผลิตไม่ได้ประสิทธิภาพ

ตัวอย่าง:
- ผู้สูงอายุที่อาศัยคนเดียว 60-75 ปี
- แม่บ้านที่ดูแลพ่อแม่
- รพ.สต./รพ.ชุมชนที่ดูแลผู้สูงอายุในชุมชน
```

### c) ทำไม Edge AI ถึงเหมาะกับปัญหานี้? (ทำไมไม่ใช่ Cloud)
```
ความเร็วในการประเมินผล งบประมาณ ไม่มีเน็ต 

เช่น: ต้องการ privacy / ไม่มีเน็ตในชนบท / ต้อง response < 1 วินาที
```

## 🚧 7. Risk & Mitigation

### Top 3 ความเสี่ยง

| ความเสี่ยง | ผลกระทบ | วิธีแก้ |
|---|---|---|
|บอร์ดพัง|ไม่สามารถอ่านค่าข้อมูลเซนเซอร์ได้|ดึงข้อมูลจากกรมอุตุนิยมวิทยา|
|ไม่มีอินเทอร์เน็ต|ไม่ได้ข้อมูลที่กว้างเเละหลากหลาย|ใช้ local data เเทน|
|เว็บไม่สามารถวิเคราะห์ภาพได้ |ไม่มีข้อมูล|ถ่ายใหม่|

ตัวอย่าง:
| ความเสี่ยง | ผลกระทบ | วิธีแก้ |
|---|---|---|
| Model accuracy drop ในสภาพแสงต่ำ | ใช้จริงไม่ได้ | เก็บข้อมูล low-light เพิ่มในรอบถัดไป |
| Servo ไม่ทำงานบน UNO Q | ไม่มี output | เตรียม Pixels เป็น backup |

## 💡 Tips สำหรับช่วงต่อยอด

1. **ถ้าต้องอธิบายงาน 2 นาที ทีมควรแบ่งช่วงพูดกันล่วงหน้า**
2. **ถ้าต้องโชว์สด ควรมีแผนสำรองเมื่อ internet หรือ hardware มีปัญหา**
3. **Document ทุก iteration** ใน GitHub — commit ทุก hour
4. **Token = constraint ที่สอนการตัดสินใจ** — เลือกสิ่งจำเป็น
5. **ไอเดียดี + execute ไม่จบ < ไอเดียเรียบง่าย + execute เสร็จ**
